Class Details

This page contains a summary of most of the courses I have taken at the University of Minnesota.

CS-5115 User Interface Design

Projects: Tabletop Manager
Keywords: Android Studio, Java, xml, User testing, group, Github

This class focused on the process of designing a user interface for a program. We came up with a concept, asked users about features they would want, analyzed the data, created the application, and then tested the program with actual users. This course had us work in a group of 5 members and we created a working program at the end. I currently have the program installed on my smartphone.

Tabletop Manager

CS-4131 Internet Programming

Projects: XKCD password generator, API mashup, Word Search
Keywords: HTML5, CSS3, Javascript, Python, Flask, SQL, Webpage, database, partner, Github

This class had us create websites using HTML, javascript, and the flask web framework. This course involved creating a number of applications, using a number of different languages and features. The majority of them have been connected to this site and are available as example programs.

Word Search Microblog Site XKCD Password generator API Mashup

CS-4511 Introduction to Artificial Intelligence

Projects: Eternal Jukebox analysis
Keywords: Python, Javascript, HTML, AI, First Order Logic, Propositional Logic, Constraint Satisfaction Problems, Search Problems, partner

We learned the elements of AI in computer systems with search algorithms, decision trees, and First Order Logic. At the end of the course we chose a relevant project, our being examining the functionality of the Eternal Jukebox Program.

Eternal Jukebox Analysis

CS-5801 Software Engineering I

Projects: TAPS User Requirements, TAPS System Specifications
Keywords: Documentation, Requirements, Specifications, UML Diagram, group, OO programming

In this class we learned about part of the procedure that companies go through when developing software, particularly the planning stages. During the course the entire class was given the task of creating the User Requirements and Design documents for a program called TAPS. In addition we talked about the Waterfall and Agile design methodologies, defining the good and bad parts of both. Risk assessment was examined and then different types of testing was discussed.

TAPS Documentation

CS-4061 Introduction to Operating Systems

Projects: Chat Client, Linux Shell Commando
Keywords: Linux, C Language, shell, processes, asynchronous processes, Makefile, partner, Valgrind

This class looked at the lower levels of programming and how to interact with the operating system of a computer. The programs were written in the C langauge and made use of many Linux commands to function. The first project of the course was a shell program which can be run over the command line, allowing the user to run other processes, but also get feedback and information not normally tracked by the command line. The second program made use of remote connections as a chat server along with client programs which could connect to said server.

Commando Shell Program Server Chat Program

CS-3081 Program Design and Development

Projects: Flashphoto Imaging Application
Keywords: C++, Libraries, Makefile, Github, OO programming, Valgrind, UML Diagram

This course went over the process of coding and building a program. We were tasked with coding the filters for the paint program. Using design patterns discussed in class we planned out how the filters connected together and how best to add them into the program. Using a UML diagram we mapped the program and the connections between the objects. After creating the filters we were asked to add command line function to the program and create tests which ensured that the program functioned correctly.

Flashphoto Imaging Application

CS-4611 Programming in Graphics and Games

Projects: Processing, Car Soccer, Earthquake, Shading, Dance
Keywords: Processing, C++, Visual Studio, openGL, Github

This class had us work on animated and sometimes interactive visual projects in 2D and 3d. The Text Rain project uses a camera to get video, and then add falling letters which get stopped when they hit dark objects. Earthquake displays a globe which shows the locations of all earthquakes around the world about when they happened chronologically. The map can be displayed in either flat or spherical modes.
