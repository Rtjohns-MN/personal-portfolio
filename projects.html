User Requirements Design Document

The Teaching Assistant Processing System (TAPS) documentation was created in the CS5801 Software Engineering 1 course. During the course I was assigned to a group and we were tasked to create the documentation to encompass the TAPS user requirements and the design specifications. We did not create the actual program, just the documentation. The User Requirements document describes the information given from the user converted into the Functional Requirements. It also details the cases for using the program maps them onto interaction diagrams. The Design Document explains the Business Layer Module (BLM) part of the program which connects the user interface and the database of the system. The document details all of the functions which the BLM uses to communicate with the UI, database and internally when processing information.

The TAPS program is designed to simplify the process and organization of managing the teaching assistants applying for positions at the University of Minnesota. The system consists of students who are applying to be TAs, professors who need TAs and the management who are assigning students for the positions. This defines the three main types of users: Applicants, Faculty and Administrators. Each of these three users need to be able to perform their own major tasks. The applicants must be able to create and manage their applications. The Faculty must be able to post positions which they need TAs for. The Administrators must be able to look at open TA positions from the faculty and then search for appropriate applicants who can fill those positions.

Eternal Jukebox Analysis

The Eternal Jukebox was created by Paul Lamere. The site allows users to search for a song using the Spotify web API, which also separates the song into segments and compares the similarity of each segment. Eternal Jukebox then reassembles the segments back into the song, but with connections between each matching segment. While playing the song when it hits a segment with a connecting segment there is a chance that it will follow the connection and continue at the new location. This allows the song to effectively remix itself dynamically to create an infinite length track.

We analyzed the program to find the best single loop location in all songs. Eternal Jukebox compares the various parts of the track to find the compatable loop sections. We adjusted the threshold for the loops to only allow loops at the end and beginning and then only high quality ones.
